# 클라우드 기반의 공간정보 데이터 통합 및 융복합 활용체계 구축(2차)

## 개요

- **기간:** 2021.06 ~ 2022.04
- **유형:** 회사 프로젝트
- **인원:** 미제공
- **역할:** SaaS(공간정보 융·복합 활용체계)팀 파트장
- **링크:** [K-GeoPlatform](https://kgeop.go.kr/)

## 프로젝트 소개

> 국토부 K-GeoPlatform 클라우드 기반 구축 사업 2차

공간정보 데이터의 통합 및 융복합 활용체계를 클라우드 기반으로 구축하는 국토교통부 사업의 2차 구축 프로젝트입니다. 구 토지대장, 비법인, 온라인 조상땅 찾기 등 다수의 서비스를 MSA 아키텍처 기반으로 개발 및 개선하였습니다.

## 주요 기능

### 구 토지대장 (WAS, MSA 구축)
- LEADTOOLS를 활용한 이미지 처리 모듈화 및 경량화, 파일 관리 체계 개선
- LEADTOOLS 이미지 보안 솔루션 업체와의 개발 협업으로 보안성 강화
- 17개 시도 코드별 테이블의 반복 Join 문제를 파티션 테이블 구조 변경으로 쿼리 성능 및 처리 속도 개선
- PostGIS를 이용한 공간 쿼리로 좌표 추출 및 PNU 초과 정확한 위치 정보 제공
- 반복 쿼리로 인한 성능 저하를 함수 적용으로 개선
- 34개 프로세스 및 21개 UI 개선 작업

### 비법인 (WAS, MSA 구축)
- DM을 활용한 데이터 시각화
- Stored Procedure를 활용한 ETL 처리 속도 및 데이터 저장 로직 간소화
- 30개 프로세스 및 15개 UI 개선

### 온라인 조상땅 찾기
- 대법원 연계(SDK)를 통한 PDF 형태의 증명서 자동 발급 시스템 개발
- 개인정보 보호를 위한 파일 암호화 프로세스 개발 및 일정 기간 파일 보관 시스템 구현

### 공통
- Spring Framework 기반 Java RESTful API 및 웹 애플리케이션 개발 및 운영
- MSA 아키텍처 기반 시스템 개발
- Oracle, PostgreSQL, Tibero 데이터베이스 Migration 작업
- DW 및 DM 구축
- RESTful API 문서화 및 TDD 단위/통합 테스트 증적 자료 작성

## 내가 기여한 부분

- SaaS팀 파트장으로서 구 토지대장, 비법인, 온라인 조상땅 찾기 서비스 전반 개발 주도
- 파티션 테이블 구조 개선을 통한 쿼리 성능 최적화 (17개 시도 코드 반복 Join 해소)
- PostGIS 공간 쿼리를 활용한 위치 정보 정확도 향상
- Stored Procedure 기반 ETL 처리 로직 설계 및 간소화
- 대법원 SDK 연계 증명서 자동 발급 시스템 및 파일 암호화 프로세스 개발

## 성과 / 배운 점

- MSA 아키텍처로 마이크로서비스 단위 프로젝트 관리 및 개선
- 데이터 동기화 및 Migration 기술 향상
- API 서버 안정적 제공 및 유지 관리
- ETL 프로세스 안정화 및 로직 표준화
- 쿼리 성능 지속 개선 및 최적화
- 파트장으로서 Kafka 기반 Migration 도입을 시도했으나, 팀 부담과 데드라인을 고려해 중단하는 판단을 내림
- ETL 프로세스 설계 및 구축 기회를 통해 안정적 개발 수행
