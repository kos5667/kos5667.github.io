# 카카오 고객센터 상담톡 플랫폼 고도화 및 운영

## 개요

- **기간:** 2022.07 ~ 현재
- **유형:** 회사 프로젝트
- **인원:** 1명
- **역할:** 기여도 100%

## 프로젝트 소개

> 카카오 사용자 상담을 처리하는 실시간 상담톡 플랫폼의 서버 개발, 운영 및 고도화 담당

카카오 사용자 상담을 처리하는 실시간 상담톡 플랫폼의 서버 개발, 운영 및 고도화를 담당했습니다. 실시간 메시징/상담 배분/통계/배치/UX 개선까지 포함한 대규모 실서비스로 고가용성, 무중단 운영이 핵심 요구사항인 환경에서 안정성과 확장성을 지속 개선했습니다.

## 목적

- 대규모 실시간 트래픽 환경에서 메시지 처리의 안정성과 지연 최소화
- Multi IDC 기반 HA 구성으로 장애 시에도 서비스 연속성 확보
- 운영 효율을 높이기 위한 배포 단위 분리 및 기술 부채 해소
- 운영 표준화로 대응 리드타임 단축, 온보딩 비용 절감

## 주요 업무

- Node.js/TypeScript 기반 실시간 상담 서버 및 Batch 개발, 운영
- Spring Boot 기반 수신 서버 운영, RabbitMQ 메시징 파이프라인 관리
- WebSocket(Socket.io) 기반 실시간 통신 구조 고도화(세션/라우팅 포함)
- Kubernetes 기반 배포 구조 설계, CI/CD 자동화 및 운영 고도화
- Redis 캐시/세션 라우팅/분산락으로 DB 병목 완화 및 I/O 최적화
- 운영 관측성 강화(로그/알림/재처리 프로세스 포함), 장애 대응 및 핫픽스 수행

## 문제 해결 과정

- Multi-Container Pod 구조로 인해 리소스 낭비 및 배포 제약 발생 → FE/BE 분리 후 Single-Container Pod로 전환, 배포 단위 분리 및 독립 확장 구조로 개선
- Multi IDC 도입 이후 GSLB 트래픽 편중 이슈 발생 → DNS 캐싱 특성과 Ingress 구성을 재점검하고, Kubernetes 레벨 2차 트래픽 제어를 추가해 분산 안정화
- 실시간 메시지 순서/중복/유실 케이스 발생 → 프로토콜 제어 로직을 도입해 중복/순서 보장, 재전송/유실 케이스를 운영 가능한 형태로 정리
- 대량 Domain 카테고리 초기화 로직이 병목 → 재귀 기반 처리 제거, BFS 구조로 전환해 초기화 성능 4배 개선
- 어뷰징/비정상 요청 유입 증가 → 정상 사용자 영향 최소화를 목표로 Rate Limit 정책을 세분화하고 분산 상태 공유 기반으로 차단 로직을 정리

## 성과 / 결과

- Multi IDC 환경에서도 **무중단 운영** 가능한 구조 확보, 장애 대응력 강화
- 트래픽 급증 시에도 안정적인 메시지 처리 및 응답 속도 유지(수평 확장 용이)
- 배포 단위 분리로 빌드/배포 시간 단축, 핫픽스 및 장애 대응 리드타임 감소
- 카나리/블루그린 배포 전략을 검증 가능한 운영 기반 확보
- 초기화 로직 최적화로 처리 성능 개선(**4배**), 운영 병목 제거
- 어뷰징 대응 체계 정비로 서비스 안정성 및 운영 품질 개선
