# CRMS 통계 시스템 개편 및 성능 최적화(통합)

## 개요

- **기간:** 2022.07 ~ 2022.11
- **유형:** 회사 프로젝트
- **인원:** 1 ~ 4명
- **역할:** 기여도 60%

## 프로젝트 소개

> CRMS 통계 서비스의 조회 성능 저하와 수치 정합성 이슈를 해결하기 위해 통계 산식, 쿼리, 배치 파이프라인 전 구간을 개편하고 성능을 최적화

CRMS 통계 서비스에서 대용량 데이터 누적에 따라 조회 성능 저하와 수치 정합성 이슈가 반복적으로 발생했습니다. 통계 산식, 쿼리, 배치 파이프라인 전 구간을 점검해 병목을 제거하고, 파티션/인덱스 전략과 쿼리 튜닝으로 성능을 개선했습니다. 외 데이터 복구, 추출 운영 이슈 등을 통합적으로 수행하였습니다.

## 목적

- 대용량 통계 조회 성능 개선 및 DB 부하 안정화
- 통계 수치 정합성 확보, 재발 방지 구조화
- 누락/유실 데이터 복구 체계 마련으로 운영 리스크 감소
- 플랫폼 이관 및 업그레이드로 운영 안정성과 확장성 확보

## 주요 업무

- 상담원/그룹/카테고리 단위 통계 로직 및 쿼리 전면 점검, 병목 구간 개선
- 파티션/인덱스 설계 및 적용, 대용량 쿼리 튜닝(QueryDSL 포함)
- 통계 산식/배치 로직 분석 및 정합성 이슈 대응(처리율, 배치 현황, 상담원 통계 등)
- 과거 데이터 누락/유실 분석 및 수동 배치 개발로 데이터 복구
- 통계 UI 개선 및 다운로드(Excel) 오류 수정(DateRangePicker, 항목 오차 등)
- 플랫폼 이관(오케스트레이션 등), 세션 클러스터링/스케줄 락 검증
- Spring Boot 업그레이드 및 javax → jakarta 전환

## 문제 해결 과정

- "성능 저하"를 단순 튜닝으로 끝내지 않고 데이터 증가 패턴과 실행 계획을 기준으로 병목을 분리, 파티션/인덱스 전략으로 스캔 범위를 줄이고 복잡 쿼리는 구조 자체를 단순화
- "수치 불일치/누락"은 산식, 쿼리, 배치 전 구간을 같이 추적해 원인을 분리, 단건 대응이 아니라 재발 방지 관점으로 검증 포인트와 복구 절차를 정리
- 이관/업그레이드는 호환성 이슈를 사전에 식별하고 단계 적용으로 운영 리스크를 통제
- 세션 클러스터링/락 검증을 통해 다중 인스턴스 환경에서 중복 실행을 방지

## 성과 / 결과

- 대용량 조회 성능 개선으로 응답 안정화, DB 리소스 사용량 감소
- 통계 정합성 이슈의 반복 발생 빈도 감소, 운영 신뢰도 개선
- 누락 데이터 복구 체계 확보로 데이터 유실 리스크와 수작업 비용 감소
- 플랫폼 이관 및 업그레이드로 장기 운영 기반 강화(확장성/안정성 확보)
- UI/다운로드 오류 개선으로 사용자 불편 및 운영 문의 감소
